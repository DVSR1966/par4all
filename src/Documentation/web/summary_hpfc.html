HPFC: Summary of features

Last update: $Date$, .

HPFC is a prototype High Performance Fortran Compiler which is being developed within the PIPS project by Fabien COELHO.

Input language

  • Fortran 77 (PIPS restriction)
  • static and dynamic mapping directives (but no array syntax)
  • independent and dynamic directives
  • assumes a number of processors known at compile time
  • distributed arrays cannot be passed as arguments (but commons can be used)

Output language

  • host and SPMD node model
  • Fortran 77
  • runtime support library
  • PVM 3.3

Implementation

  • data structures based on NEWGEN
  • polyhedron manipulations based on the Linear/C3 library
  • 17,000 lines of C for the compiler (HPFC, a PIPS module)
  • 1,000 lines of shell and C for the driver and other utilities
  • 3,000 lines of m4/fortran for the runtime support library (expanded to +8,000)
  • 2 programs are generated (host and SPMD node) and linked to PVM and the HPFC runtime library.

Standard Optimizations

  • on top of a basic run-time resolution scheme:
  • reduced allocation of distributed arrays on nodes.
  • overlapping accesses of block distributed stride-1 aligned arrays (overlapping declarations, vector messages, iteration set...)
  • subarray shifts of block distributed stride-1 aligned arrays
  • reductions on block distributed stride-1 aligned arrays (by name)

Advanced Optimizations

  • linear algebra-based techniques...
  • I/O-related communications (abstract, paper)
  • remapping-related communications and optimizations (abstract, paper)

Experiments

  • The compiler is validated on +100 small or medium size codes for non regression testing every night.
  • SOR on a network of SUN workstations (abstract, paper)
  • wave propagation code from IFP on the SEH/ETCA CM5: problems with arithmetic exceptions and PVM (was 3.3.2).
  • matrix transpose on a DEC Alpha farm.

Future work

  • implementation of a linear algebra based technique (abstract, paper)
  • more experiments
